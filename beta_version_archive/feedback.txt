개선점

1. 간헐적 통신 에러
단일 PC에서 멀티스레딩을 통해 다수의 사용자를 통한 소켓통신을 할 경우 문제가 없었음
하지만 복수의 PC를 사용할 경우, 간헐적으로 소켓이 닫히는 현상이 발생했음

2. 버튼을 클릭했을 때 직관성 부족
버튼을 눌렀을때, 입력이 들어갔다는 효과(회색처리, 이미지 테두리 볼드 처리) 등을 넣지 못했다.
중복 입력 가능성이 생김


발전

1. Application 화
아무래도 Application 으로 구현한다면 지금보다 훨씬 더 사용하기 좋을 것
실제 같이 진행해보려고 했으나 이석재 학생의 능력부족으로 실패

배운 점

1. 네트워킹 기능을 이용한 최초의 텀 프로젝트
2. 좋은 아이디어를 잘 구현하는 것이 매우 힘들다.
(GUI, 내부 로직, 테스팅)
3. 그래서 실제 프로그램 개발에 들어간 시간이 다른 조에 비해 늦었는데,
더 늦어 지기 전에 최소 기능(소켓통신, 승자연전제)에 집중을 해서 전체적인 틀을 만들고,
기능을 체크하면서 그 후에 필요한 추가 알고리즘, 기능 등을 붙여나가면서 완성했다.
(Agile, Lean startup, MVP)
